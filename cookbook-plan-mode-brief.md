# Cookbook App — Plan-Mode Brief

A local, single-device web app for me and my partner to save the dishes we've cooked, get inspiration when we don't know what to make, and run timers while cooking. No backend, no sign-in, no AI at runtime.

---

## Core principles (do not violate without asking me)

- **Web app, not native.** Vite + React, plain JavaScript (no TypeScript).
- **No backend, no auth, no server.** Everything runs in the browser.
- **Storage:** IndexedDB (not localStorage) because dishes include photos. Use a small wrapper library (`idb`) so the code stays readable.
- **No AI / no API calls at runtime.** The only "Claude" integration is a copy-to-clipboard button + a plain link to the Claude web app. Nothing is generated by the app.
- **Single device for now.** Data lives in one browser. Do NOT add sync, accounts, or a server. If syncing ever comes up, it's a separate future phase — flag it, don't build it.
- Keep dependencies minimal. If a feature can be done with built-in browser APIs, do that instead of adding a library.

---

## Data model

Each **dish** has:

- `id` (generated)
- `name` (string)
- `origin` (array of tags — e.g. `["Japanese"]`, `["Chinese", "Taiwanese"]`, `["Korean", "American"]`. A dish can have multiple. Filterable. Keep a controlled-ish list of common tags but allow new ones.)
- `timeCategory` (one of: `fast`, `medium`, `time-consuming`)
- `ingredients` (array of strings)
- `appliances` (array of strings — e.g. air fryer, cast iron pan, steamer)
- `steps` (array of strings, ordered)
- `photos` (array of image blobs stored in IndexedDB; can be empty)
- `timers` (optional array of `{ label, seconds }` — e.g. `{ label: "Air fry", seconds: 720 }`) so a dish can define one or more named timers

Seed data: I will provide my existing dish list. Load it on first run if the database is empty. After that, the "Add dish" form is the source of new dishes.

## Screens / features

### 1. Home
The landing screen — see the **Home screen** section below for the full spec. One big question, two options: **"no idea"** (gacha draw) and **"gimme inspo"** (browse menu).

### 2. "no idea" — gacha draw (random mode)
- Reached from the home screen's "no idea" option.
- **Function first:** pick one random dish and show it. This must work as a plain instant result before any animation is added.
- **Animation (layer on after the plain draw works):** a Japanese capsule-toy (gachapon) sequence —
  1. A hand turns the machine's crank.
  2. A capsule ball drops out.
  3. The ball opens.
  4. A slip of paper unfurls from inside, showing the dish **name** and a dish **image**.
- Build this with simple SVG + CSS animation. **No animation libraries** unless one is genuinely needed and you ask me first.
- A clear **"Redraw"** that runs the draw again (the animation can be quicker/skippable on redraw).
- Tapping the result opens **Dish detail** (shared screen).
- Optional, only if trivial: let me limit the draw to a time category before rolling. Skip if it adds real complexity.

### 3. Browse menu
- Scrollable list of all dishes.
- **Filters:** by time category (fast / medium / time-consuming) and by origin. A dish can have multiple origin tags, so a dish matches an origin filter if *any* of its tags match (e.g. a `["Chinese", "Taiwanese"]` dish shows under both "Chinese" and "Taiwanese").
- Each list item shows name, origin tag(s), time category, and a thumbnail if a photo exists.
- Tapping opens **Dish detail**.

### 4. Dish detail (shared by both modes)
- Dish name, origin tag(s), time category.
- **Ingredients** as a checklist — I can tick off items I already have. Ticked state is just UI for this session (does not need to persist), unless persisting is trivial.
- **Appliances needed** — a simple list.
- **Photos** — show the dish's photo(s) as framed objects on the cream ground; if multiple, a simple swipe/scroll. Any illustrated/hand-drawn images are just uploaded photos; the app does not generate them.
- **Steps** — numbered instructions.
- **Timers** — for any timer defined on the dish (e.g. steak, air-fried items), show a start button. See Timer behavior below.
- **Ask Claude for more** — two controls:
  1. A **"Copy prompt"** button that copies this exact text to the clipboard: `give me a detailed recipe and cooking instructions for [dish name]` (with the real dish name filled in).
  2. A **"Open Claude"** link/button that opens `https://claude.ai/new` in a new tab.
  - So the flow is: one click to copy, one click to open Claude, then I paste. Do not call any API.

### 5. Add dish
- A form to add a new dish with all the fields in the data model: name, origin (allow selecting/adding **one or more** tags), time category, ingredients (add multiple), appliances (add multiple), steps (add multiple, ordered), photo upload (one or more), and optional named timers.
- On save, write to IndexedDB and return to the menu.
- Nice-to-have: edit/delete an existing dish. Build add first; add edit/delete only after add works.

### 6. Timer behavior
- A timer is a countdown started from a dish step or the dish's timer list.
- Show remaining time clearly (mm:ss).
- When it hits zero, alert me — a sound and/or a visible flash. Browser audio + a vibration on mobile if available.
- Should keep running while I scroll the steps. Multiple timers at once is a nice-to-have, not required for v1 — if it's hard, allow one at a time and say so.

## Design language
Reference: cabbages.world recipe pages (I have reference screenshots). Follow this closely from the start — it's specific.

- **Ground:** warm off-white / cream paper, not dark, not stark white. This is the dominant surface everywhere.
- **Display font:** a chunky pixel / bitmap font for dish names, headings, and the big home question. Use a free pixel font close to the reference (e.g. something in the family of "Pixelify Sans", "Silkscreen", or similar via Google Fonts) — pick the closest clean free option. Do not buy a font.
- **Body font:** clean, plain sans-serif for descriptions, ingredients, and instructions. Highly readable, neutral.
- **Labels & links:** underlined monospace treatment for small labels and links (e.g. "Ingredients:", "Instructions:", "Watch the video" → here it'll be the Claude link). Matches the reference's underlined-mono accents.
- **Layout:** generous whitespace, editorial. Thin horizontal rules separating sections. Ingredients shown as floating photo objects on the cream ground, each with a small caption underneath (quantity + name), exactly like the reference.
- **Restraint:** muted, mostly monochrome-on-cream with the photos providing the color. No heavy UI chrome, no cards-everywhere. Let the paper ground and type breathe.
- I'll give design feedback as it comes together. Get the cream ground, pixel headings, and ingredient-object layout right first — those three carry the whole look.

- Two options below:
  1. **"no idea"** → the gacha draw (see Random mode).
  2. **"gimme inspo"** → the Browse menu + filters.

## Ingredient & appliance imagery (photo cut-outs, not line-art)
Changed from the earlier icon plan: the look uses **photographic cut-out objects on cream**, not generated SVG icons.

- **Do NOT generate SVG icons for ingredients.** Display uploaded photos instead.
- **v1 approach (keep adding-a-dish easy):** display whatever photo is provided in a consistent framed style on the cream ground (e.g. uniform sizing, subtle rounding) so it reads as the reference look *without* requiring a perfect background-removed cut-out for every ingredient. Background removal must never be required to add a dish.
- **Reusable common-ingredient library:** maintain a small set of background-free PNGs for the most common ingredients (egg, garlic, onion, scallion, carrot, rice, noodles, chicken, beef, pork, fish, tofu, butter, soy sauce, etc.). When a dish uses one of these, reuse the library image automatically so I don't re-upload. Map ingredient name → library image, with graceful fallback to the uploaded photo or a plain caption if neither exists.
- Appliances: same approach — a small reusable set, displayed as framed photo objects, text fallback if none.
- Perfect hand-cut-out images are a "nice to have when I feel like it," never a requirement. Never block a dish on imagery.
